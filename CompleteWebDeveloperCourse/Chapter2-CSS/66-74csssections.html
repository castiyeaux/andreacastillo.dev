Borders

  1. The three boxes above were made completely with CSS via the <style> tag in the head - no HTML code.
    1. (...except for...) The <div> tags house the general square class and the individual color ids (square-red, square-blue, and square-green).
  2. The individual color ids have the background-color and border properties.
    1. The border property has three defined values listed below (in any order) and acts like the margin property in that it can have the below as separate, specifically listed properties:
      • border-width, in pixels
      • border-color, either standard name, hexidecimal value, or RGB notation
      • border-style, from none, hidden, dotted, dashed, solid, double, groove, ridge, inset, outset, and inherit
  3. The square class lists all of the common properties between the three squares:
    • width, square dimension
    • height, square dimension
    • margin, to set the space around/in between each square (otherwise they would be flush next to each other
    • float, set to "left" to get them all horizonally aligned
  4. To keep this ordered list from interfering with the squares, a fourth <div> was created with the clear-div id.
    • clear: both;
  5. Per #2a, the blue square's borders are showcasing how each side can be a different border, with the same potential list order as margins and padding (top, right, bottom, left):
    • border-style: ridge solid dotted dashed;
    • border-color: red yellowgreen orange orchid;
    • border-width: 7px;
  6. border-radius doesn't actually add a border. It adds a radius around the corners, which can be adjusted via pixels, to give it a more rounded look. Can even make it a circle (by setting radius to half the square's width, or '50%' to avoid constant math)!
    • border-radius: 50px;
