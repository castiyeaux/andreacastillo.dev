* Main difference between relative position and margin:
- Relative position is manipulating the position by basically adding a margin, but only on one side at a time. Ex. left: 100px; top: -100px;
- Margins add "padding" on all sides of the object which, in turn, manipulate the position.

Margins are used when you want the element to remain in the flow of the page, and to spread everything out relative to it.
Positioning is used when you want the element to be taken out of the flow of the page, and move it around on top of other elements.

Padding

"Padding is pretty similar to margins, but it works on the inside of an object, rather than the outside."

Paragraph with 5px gap (padding) between text and blue div.

Paragraph with 70px top padding, 10px bottom padding, 30px left padding, and 50px right padding.

* If there is a width previously specified (as with all the above examples), the padding is added on top/in addtion to the width.
- Ex. The blue box above has a width of 200px. Plus the 30px left padding and 50px right padding, total/actual width of the blue box is 280px.

--

Browsers have default margins and padding built-in to their elements - which can make the #pixelperfect design difficult to get.

To get rid of the defaults:
'''
body {
margin: 0;
padding: 0;
}
p {
margin: 0;
padding: 0;
}
'''

* Do 'p' if needed! i.e. if there is still a gap in between the boxes.

* Since the default margins and padding can vary from browser to browser, using a CSS Reset can help reset the default - no matter the browser - to help match the output to your input. (i.e. If your site looks different on different browsers, despite the same exact code)
